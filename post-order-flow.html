8. order.addItem(item) Kya Karta Hai?

Ye method Order entity me hai. Ye bahut important hai.

44: @OneToMany(mappedBy = "order", cascade = CascadeType.ALL, orphanRemoval = true, fetch = FetchType.LAZY) 45: @Builder.Default 46: private List<OrderItem> items = new ArrayList<>(); 48: public void addItem(OrderItem item) { 49: items.add(item); 50: item.setOrder(this); 51: }
  • Line 44: One order ke many order items ho sakte hain.
  • cascade = CascadeType.ALL means order save karoge to items bhi save honge.

  • Line 50: item ke andar parent order set hota hai. Isse order_items table me order_id set ho paata hai.

8A. @OneToMany Line Detailed Walkthrough

Is single annotation me parent-child relation, save behavior, delete behavior, aur loading behavior define hota hai.

@OneToMany(mappedBy = "order", cascade = CascadeType.ALL, orphanRemoval = true, fetch = FetchType.LAZY) @Builder.Default private List<OrderItem> items = new ArrayList<>();
Part Simple Meaning Why Needed Here
@OneToMany One Order ke andar many OrderItems ho sakte hain. Ek customer order me Laptop, Mouse, Keyboard jaise multiple items ho sakte hain.
mappedBy = "order" Relation ka owner OrderItem class ka order field hai. Foreign key order_id child table order_items me hoti hai, parent table me nahi.
cascade = CascadeType.ALL Parent Order par save/update/delete operation child items tak cascade ho sakta hai. Hum sirf orderRepository.save(order) call karte hain; items automatically save ho jaate hain.
orphanRemoval = true Agar item order ki list se remove hua, to wo database se bhi delete ho sakta hai. Order ke bina OrderItem ka meaning nahi hai, so orphan item ko clean karna correct hai.
fetch = FetchType.LAZY Order fetch karte waqt items immediately load nahi honge. Performance better hoti hai. Items tab load honge jab order.getItems() call hoga.
@Builder.Default Lombok builder use karte time default empty list preserve karta hai. Without this, builder se Order banane par items null ho sakta tha, aur addItem par NullPointerException aa sakta tha.
new ArrayList<>() Items list initially empty hai, null nahi. New order banate hi items safely add kar sakte hain.
Easy memory trick: @OneToMany relation batata hai, mappedBy owner batata hai, cascade save behavior batata hai, orphanRemoval delete behavior batata hai, aur LAZY loading behavior batata hai.

8B. mappedBy = "order" Ko Deeply Samjho

Ye confusing part hota hai. Is project me relation bidirectional hai: Order ko items pata hain, aur OrderItem ko parent order pata hai.

Parent Side: Order.java

@OneToMany(mappedBy = "order", ...) private List<OrderItem> items;

Order ke paas child items ki list hai.

Owner Side: OrderItem.java

@ManyToOne(fetch = FetchType.LAZY) @JoinColumn(name = "order_id", nullable = false) private Order order;

OrderItem table me actual foreign key order_id hoti hai.

mappedBy = "order" me jo "order" likha hai, wo database column name nahi hai. Wo OrderItem.java ke field ka naam hai: private Order order;

8C. addItem Method Detailed Walkthrough

48: public void addItem(OrderItem item) { 49: items.add(item); 50: item.setOrder(this); 51: }
Line Meaning Why Important
48 Helper method starts. It accepts one OrderItem. Service code clean rehta hai: order.addItem(item).
49 Item parent order ki list me add hota hai. Now parent object knows: mere paas ye child item hai.
50 Child item ke andar parent order set hota hai. Now child object knows: mera parent order kaun hai. Isi se order_id set hota hai.
51 Method ends. Parent-child relation dono side se complete ho gaya.
Sirf items.add(item) enough nahi hota. Database foreign key child side par hoti hai, so item.setOrder(this) required hai.

14. Scheduler Use Case

Scheduler ka kaam hota hai background me automatic task run karna. User ko API call karne ki zarurat nahi hoti.

Is project me scheduler every 5 minutes PENDING orders ko PROCESSING me move karta hai.

Real Life Example

  • Customer order create karta hai.
  • Order ka status PENDING hota hai.
  • Warehouse/system kuch time baad order processing start karta hai.
  • Scheduler automatically PENDING ko PROCESSING bana deta hai.

Why Scheduler?

  • Manual API call ki need nahi.
  • Repeated work automatic ho jata hai.
  • Background processing cleanly service layer me rehta hai.
  • Future me email, cleanup, retry jobs bhi add kiye ja sakte hain.

Line Meaning

14 Scheduler service ko call karega. Actual business logic scheduler me nahi, service me hai.

16. Cron Expression Explained

@Scheduled(cron = "0 */5 * * * *")
Part Value Meaning
Second 0 Minute ke 0th second par run karo.
Minute */5 Har 5 minute me run karo.
Hour * Har hour me allowed.
Day * Har day me allowed.
Month * Har month me allowed.
Weekday * Har weekday me allowed.
Ye scheduler 12:00:00, 12:05:00, 12:10:00, 12:15:00 jaise times par run karega.

19. Save Ke Time Database Me Kya Insert Hota Hai?

Jab orderRepository.save(order) call hota hai, Hibernate roughly ye kaam karta hai.

-- 1. Parent order save hota hai insert into orders (customer_id, total_amount, status, created_at, updated_at) values (1001, 72000.00, 'PENDING', now(), now()); -- 2. Database generated order id maan lo 1 hai -- 3. Child items save hote hain with order_id = 1 insert into order_items (product_name, quantity, price, order_id) values ('Laptop', 1, 70000.00, 1); insert into order_items (product_name, quantity, price, order_id) values ('Mouse', 2, 1000.00, 1);
Ye child item inserts isliye automatically hote hain kyunki relation me cascade = CascadeType.ALL laga hua hai.

20. Status Kaise Maintain Ho Raha Hai?

Status ek enum hai. Order entity me status field database me string ke form me save hota hai.

32: @Enumerated(EnumType.STRING) 33: @Column(nullable = false, length = 20) 34: private OrderStatus status;
  • @Enumerated(EnumType.STRING) means enum value database me text ke form me save hogi.
  • Example: PENDING, PROCESSING, SHIPPED, DELIVERED.
  • nullable = false means status empty/null nahi ho sakta.
Jab order create hota hai, service line 51 par status set karti hai: .status(OrderStatus.PENDING). Isliye every new order starts as PENDING.

21. Status Lifecycle Code Me Kaise Enforce Ho Raha Hai?

Project valid transitions map maintain karta hai. Is map se decide hota hai ki current status se next status allowed hai ya nahi.

33: // Valid transitions map: current status -> allowed next statuses 34: private static final Map<OrderStatus, Set<OrderStatus>> VALID_TRANSITIONS = new EnumMap<>(OrderStatus.class); 36: static { 37: VALID_TRANSITIONS.put(OrderStatus.PENDING, Set.of(OrderStatus.PROCESSING, OrderStatus.CANCELLED)); 38: VALID_TRANSITIONS.put(OrderStatus.PROCESSING, Set.of(OrderStatus.SHIPPED)); 39: VALID_TRANSITIONS.put(OrderStatus.SHIPPED, Set.of(OrderStatus.DELIVERED)); 40: VALID_TRANSITIONS.put(OrderStatus.DELIVERED, Set.of()); 41: VALID_TRANSITIONS.put(OrderStatus.CANCELLED, Set.of()); 42: }
Current Status Allowed Next Status Meaning
PENDING PROCESSING, CANCELLED New order process ho sakta hai ya cancel ho sakta hai.
PROCESSING SHIPPED Processing ke baad order ship hoga.
SHIPPED DELIVERED Shipped ke baad customer ko deliver hoga.
DELIVERED No next status Delivered final state hai.
CANCELLED No next status Cancelled final state hai.
Normal lifecycle: PENDING -> PROCESSING -> SHIPPED -> DELIVERED. Cancel path: PENDING -> CANCELLED.

23. validateTransition Method

Ye method actual guard hai. Ye invalid status jump ko block karta hai.

130: private void validateTransition(OrderStatus current, OrderStatus next) { 131: Set<OrderStatus> allowed = VALID_TRANSITIONS.getOrDefault(current, Set.of()); 132: if (!allowed.contains(next)) { 133: throw new InvalidStatusTransitionException(current, next); 134: } 135: }
  • Line 131: Current status ke allowed next statuses map se milte hain.
  • Example: current PENDING hai to allowed = PROCESSING, CANCELLED.
  • Line 132: Agar requested next status allowed list me nahi hai, to error throw hota hai.
  • Line 133: Invalid transition par API 400 Bad Request return karegi.
Example: PENDING se DELIVERED direct allowed nahi hai. Pehle PROCESSING, fir SHIPPED, fir DELIVERED jaana padega.

Scheduler manual API ke bina PENDING orders ko PROCESSING banata hai. Ye lifecycle ka first automatic step hai.
